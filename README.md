# Supabaseの動作確認

基本的には、非推奨。

supabaseは、エッジ関数の管理が難しいので、エッジ関数不要の、めちゃくちゃシンプルなシステムの時はいいかも。

実装例では、一応、エッジ関数も動作しているけど、偶然動いているような状態。

## エッジ関数

### 設定

認証が必要な場合は、sb/supabase/config.tomlのverify_jwtをfalseにして、認証トークンではじく。

認証を必要としない場合はtrueにして、ANONトークンではじく。

### クライアント

クライアントは、認証しているときとしていない時では、Authorizationヘッダーのトークンを自動的に切り分けてくれているみたい。

ヘッダーを自分で設定すると、動かなくなることがあった。

### エッジ関数内

エッジ関数内では、認証のAPIの時は、Authorizationヘッダーの認証情報をつけてインスタンスを作る必要がある。

ただし、service_roleのDBテーブルを操作する時は、つけると動かないので、インスタンスが２つ必要になる。

### パフォーマンス

コールドスタートは関数ごと。
